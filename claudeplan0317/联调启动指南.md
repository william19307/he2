# 联调启动指南（登录 + H5 答题）

## 一、接口访问地址（本地）

| 用途 | 地址 |
|------|------|
| **后端 API 根路径** | `http://localhost:3002` |
| **健康检查** | `GET http://localhost:3002/api/health` |
| **REST 前缀** | `http://localhost:3002/api/v1` |
| **PC 登录** | `POST http://localhost:3002/api/v1/auth/login` |
| **H5 验证** | `POST http://localhost:3002/api/v1/h5/verify` |
| **H5 任务列表** | `GET http://localhost:3002/api/v1/h5/tasks`（需 Bearer） |
| **前端开发站** | `http://localhost:5173`（Vite 会把 `/api` 代理到 3002） |

前端页面在浏览器里一般用：**`http://localhost:5173/login`**、**`http://localhost:5173/h5/verify`**，请求走同源 `/api/v1/...`，无需改 CORS。

## 二、怎么把环境开起来（按顺序）

### 1. 打开 Docker Desktop（macOS）

确保菜单栏鲸鱼图标为运行状态。

### 2. 启动 MySQL

在 **`heart-test/backend`** 目录：

```bash
# 若已有 mentalhealth-mysql 容器（曾创建过）
docker start mentalhealth-mysql

# 或首次/用 compose 起库
npm run db:up
```

确认 3306 可连（约等 10～30 秒容器就绪）。

### 3. 数据库表 + 测试数据

仍在 **`backend`**：

```bash
# 首次或改 schema 后
npx prisma db push

# 可重复执行：心理老师、学生、ongoing 计划、pending 任务
npm run db:seed
```

### 4. 启动后端 API

```bash
# 若 3002 被占用先释放
# lsof -ti :3002 | xargs kill -9

npm run start
```

看到日志里 **`http://localhost:3002`** 即成功。

### 5. 启动前端

另开终端，在 **`heart-test/frontend`**：

```bash
npm install   # 首次
npm run dev
```

浏览器访问 **`http://localhost:5173`**。

## 三、联调用测试数据（seed 后）

| 项 | 值 |
|----|-----|
| 学校编码 | `demo_school` |
| 心理教师 | `counselor001` / `123456` |
| 学生（H5） | `student001` / `123456` |
| 测评计划 | `【联调】进行中测评`，**status = ongoing** |
| 学生任务 | **student001** 对应 **students** 表记录 + 至少 **1 条** `assessment_tasks`（**pending**，PHQ-9） |

每次执行 **`npm run db:seed`** 会：保证上述账号与计划存在；若 student001 的任务已被提交，会**重置为 pending** 方便反复测 H5。

## 四、常见问题

| 现象 | 处理 |
|------|------|
| Can't reach database | Docker 未开或未 `docker start` / `npm run db:up` |
| 3002 端口占用 | `lsof -ti :3002 \| xargs kill -9` 后重启后端 |
| 前端 404 / 网络异常 | 必须用 **`npm run dev`** 打开 5173，不要只打开静态文件 |
| seed 报唯一约束 | 已改为 upsert；若仍报错请把完整日志发给后端 |

前端接口报错时，把 **请求 URL、状态码、响应 body** 或截图发给后端即可对照修复。
