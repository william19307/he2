## 一、设计系统规范

### 1.1 Arco Design 版本与引入

```bash
# 前端安装
npm install @arco-design/web-vue
# 或 React 版
npm install @arco-design/web-react
```

使用 **Arco Design Vue 2.x** 或 **Arco Design React 2.x**，均可。本文统一以组件名描述，不区分框架，工程师按实际框架引用。

### 1.2 色彩系统

在 Arco 全局 Token 基础上，覆写以下品牌色：

| Token 名 | 默认值 | 说明 |
|---------|--------|------|
| `--color-primary-6` | `#4A7C6F` | 品牌主色·鼠尾草绿 |
| `--color-primary-5` | `#6FA89A` | 主色 Hover |
| `--color-primary-7` | `#2D5449` | 主色 Active / 深色 |
| `--color-primary-1` | `#E8F2F0` | 主色极浅·背景填充 |
| `--color-danger-6`  | `#C0392B` | 红色预警 |
| `--color-warning-6` | `#D4A017` | 黄色预警 |
| `--color-success-6` | `#27AE60` | 正常状态 |
| `--color-text-1`    | `#1A2332` | 主文字 |
| `--color-text-3`    | `#7A8FA0` | 次级文字 |
| `--color-bg-2`      | `#F7F5F2` | 页面背景 |
| `--color-border`    | `#E2DED8` | 通用边框 |

> **规则**：所有颜色必须引用 Token，禁止硬编码 HEX 值。设计师在 Figma 中使用 Arco Design Kit 内置变量层。

### 1.3 字体规范

```css
/* 中文 */
font-family: "PingFang SC", "Noto Sans SC", "Microsoft YaHei", sans-serif;

/* 数字 / 英文强调 */
font-family: "DIN Alternate", "Roboto", monospace; /* 用于 KPI 数字 */
```

| 用途 | 字号 | 字重 | 行高 |
|------|------|------|------|
| 页面主标题 | 20px | 600 | 28px |
| 模块标题 | 16px | 600 | 24px |
| 卡片标题 | 14px | 500 | 22px |
| 正文 | 14px | 400 | 22px |
| 辅助文字 | 12px | 400 | 20px |
| KPI 数字 | 32px | 700 | 1 |
| 小 KPI 数字 | 24px | 700 | 1 |

### 1.4 间距系统

遵循 Arco 4px 基准网格：`4 / 8 / 12 / 16 / 20 / 24 / 32 / 48px`

| 场景 | 间距 |
|------|------|
| 卡片内边距 | `20px 24px` |
| 页面内容区外边距 | `24px 32px` |
| 表单项间距 | `20px` |
| 行内元素间距 | `8px` |
| 卡片间距 | `16px` |
| 区块间距 | `24px` |

### 1.5 圆角规范

| 元素 | 圆角 |
|------|------|
| 卡片 / 面板 | `8px` |
| 按钮 | `4px`（Arco 默认）|
| 输入框 | `4px` |
| 标签 Tag | `12px`（胶囊形）|
| 头像 | `50%` |
| 模态框 | `8px` |
| 小徽标 | `2px` |

### 1.6 阴影规范

```css
--shadow-sm: 0 1px 4px rgba(0,0,0,.06);    /* 卡片悬浮 */
--shadow-md: 0 4px 16px rgba(0,0,0,.10);   /* 下拉菜单 / Popover */
--shadow-lg: 0 8px 32px rgba(0,0,0,.14);   /* 模态框 */
```

│  Arco Layout                                                  │

### 2.2 Sider（左侧导航）

**组件**：`a-layout-sider` + `a-menu`（内嵌模式）

**宽度**：展开 `220px`，收起 `48px`，通过 `collapsed` 属性控制，顶部有收起按钮。

**结构（从上到下）**：

```
[Logo 区域] 高度 56px，与 Header 等高
  - 图标（32×32 品牌图标）+ 产品名"心晴平台"（收起时仅显示图标）

[当前用户信息块] 高度 64px
  - Avatar（32px，姓氏首字）
  - 姓名（14px 500）+ 角色标签（12px Tag）
  - 收起时仅显示 Avatar

[Menu 导航列表]
  类型：a-menu inline，accordion 模式

  ── 工作台 ──
  ■ 工作台首页           icon: icon-dashboard
  ■ 消息通知             icon: icon-notification  badge: 未读数

  ── 测评管理 ──
  ■ 测评计划             icon: icon-list
  ■ 量表题库             icon: icon-formula
  ■ 自定义问卷           icon: icon-edit

  ── 预警 & 干预 ──
  ■ 预警管理             icon: icon-exclamation-circle  badge: 待处理数（红色）
  ■ 个案管理             icon: icon-file
  ■ 干预方案库           icon: icon-heart

  ── 学生 ──
  ■ 学生档案             icon: icon-user
  ■ 家长沟通             icon: icon-phone

  ── 数据分析 ──
  ■ 班级看板             icon: icon-bar-chart
  ■ 年级看板             icon: icon-rise
  ■ 地市看板（权限控）   icon: icon-global

  ── 系统 ──
  ■ 组织 & 用户管理      icon: icon-settings
  ■ 系统配置             icon: icon-tool

[底部]
  - 版本号 v1.0（收起时隐藏）
```

**样式覆写**：
- 选中项：`background: var(--color-primary-1)`，左侧 3px 实线 `var(--color-primary-6)`
- Hover：`background: #F2F7F5`
- Badge 红色预警数：`a-badge` status=danger，置于菜单项右侧

### 2.3 Header（顶部栏）

**组件**：`a-layout-header`，高度 56px，背景白色，底部 1px border

**左侧**：`a-breadcrumb`，显示当前页面路径，最多两级

**右侧（从左到右）**：
1. 全局搜索框（`a-input-search`，宽度 220px，placeholder="搜索学生、量表、计划…"）
2. 通知铃铛（`a-badge` count=3 + icon-notification，点击打开 `a-drawer` 通知面板）
3. 帮助文档入口（icon-question-circle，Tooltip="帮助文档"）
4. 分割线
5. 用户头像 + 姓名 + 下拉菜单（个人设置 / 修改密码 / 退出登录）

### 2.4 页面内容区通用结构

每个页面内容区遵循以下结构顺序：

```
[页面标题行]
  - 左：页面 H1 标题（20px 600）+ 副标题（12px 文字3色）
  - 右：主操作按钮（a-button type=primary）

[筛选 & 搜索栏]（如有）
  - a-card（no-border padding=16px）包裹
  - 左侧一排筛选条件（Input / Select / DatePicker）
  - 右侧次要操作（导出、批量操作等）

[主内容区]
  - 数据表格 / 卡片列表 / 详情面板等

[分页]（表格页面）
  - a-pagination，右对齐，显示 total，pageSize 可选 20/50/100
```

## 四、登录 & 认证

### 4.1 登录页

**布局**：左右分栏，左侧 55% 品牌插画区，右侧 45% 表单区。响应式：小于 768px 时全屏表单。

**左侧品牌区**：
- 背景：深色品牌渐变（`#2D5449` → `#4A7C6F`）
- 中心：产品 Logo + slogan + 3 个核心功能点卡片（玻璃态效果）
- 底部：政策支持背书文字（"响应教育部学生心理健康专项行动计划"）

**右侧表单区**：

```
组件结构：
  a-form（layout=vertical）
    ┌ 产品 Logo 小版（48px 图标 + 文字）
    │
    ├ 标题："欢迎回来"（20px 600）
    ├ 副标题："请选择登录身份"（12px 文字3色）
    │
    ├ 角色切换
    │   a-radio-group（button 模式）
    │   选项：学生 / 班主任 / 心理老师 / 医生 / 管理员
    │
    ├ 学校选择（非学生角色显示）
    │   a-select（支持搜索，placeholder="请选择所在学校"）
    │
    ├ 账号输入
    │   a-input，prefix-icon=icon-user，placeholder="账号"
    │
    ├ 密码输入
    │   a-input-password，prefix-icon=icon-lock，placeholder="密码"
    │
    ├ 记住登录 + 忘记密码（同行，flex 分列）
    │
    ├ 登录按钮（a-button type=primary，long，loading 状态）
    │
    └ 底部提示（"首次登录？请联系学校管理员获取账号"）
```

**交互行为**：
- 角色切换时，"学校选择"字段按规则显示/隐藏（学生端无此项，直接用学号+密码）
- 登录失败：账号/密码错误连续 3 次，显示图形验证码（`a-modal` 内嵌滑块或图形码）
- 登录成功：`Message.success('登录成功')` → 300ms 后跳转到对应角色的默认首页

**状态清单**：

| 状态 | 表现 |
|------|------|
| 默认 | 表单空白可输入 |
| 加载中 | 登录按钮 loading，表单字段 disabled |
| 账号错误 | a-form-item 标红 + 错误提示文字 |
| 连续失败 3 次 | 弹出验证码 Modal |
| 账号被禁用 | `a-alert type=error`：该账号已被禁用，请联系管理员 |

## 五、工作台 Dashboard

> 面向心理老师/咨询师的默认首页。班主任版、管理员版布局类似但内容不同，下文标注差异。

```
[页面标题行]
  标题："工作台"
  副标题："2025-03-17，星期一，上午好，张晓慧老师"
  右侧：无主操作按钮

[KPI 指标行]（a-row gutter=16，4 等分）
  卡片1：今日待处理预警
  卡片2：本周测评完成率
  卡片3：在跟进个案数
  卡片4：本月新增预警

[内容主区域]（a-row gutter=16）
  左列（span=14）：
    · 待处理预警列表
    · 今日跟进任务

  右列（span=10）：
    · 全校心理健康概览（环形图）
    · 近期测评计划进度

[底部卡片行]（a-row gutter=16，3 等分）
  · 快捷操作入口
  · 本月预警趋势折线

```

### 5.2 KPI 卡片

**组件**：`a-card`（`hoverable`，`bordered`）

每张卡片结构：
```
┌─────────────────────────────────────┐
│  [图标 32px]  标题文字（12px 文字3）│
│                                     │
│  数字（32px DIN 700）               │
│                                     │
│  环比变化（12px）  ↑12% vs 上周     │
└─────────────────────────────────────┘
```

4 张卡片内容定义：

| 卡片 | 图标 | 数值来源 | 变化方向 |
|------|------|----------|----------|
| 待处理预警 | icon-exclamation-circle（红色） | `alerts.count(status=pending)` | 越少越好（减少为绿，增加为红）|
| 本周测评完成率 | icon-check-circle（绿色） | `tasks.completion_rate(week)` | 越高越好 |
| 在跟进个案 | icon-file（蓝色） | `cases.count(status=active)` | 中性展示 |
| 本月新增预警 | icon-rise（橙色） | `alerts.count(month)` | 中性展示 |

**班主任版 KPI**：本班完成率 / 本班黄色预警 / 本班未完成人数 / 下次测评截止日

**管理员版 KPI**：全校覆盖率 / 全校红色预警 / 全校在跟进个案 / 本月测评计划数

### 5.3 待处理预警列表

**组件**：`a-list`，`a-list-item`

每条预警 item 结构：
```
┌──────────────────────────────────────────────────────────┐
│ [Tag 红/黄]  学生姓名（14px 500）  班级（12px 文字3）    │
│ 触发原因描述（12px 文字3，单行省略）                      │
│ 触发时间（12px 文字4）                [处理] [查看] 按钮  │
└──────────────────────────────────────────────────────────┘
```

- 红色预警：左侧 4px 红色竖线 + 背景 `#FFF0EF`
- 黄色预警：左侧 4px 橙色竖线 + 背景 `#FFFAED`
- 超过 24h 未处理的红色预警：item 右上角显示 `a-badge` "超时"（红底）
- 列表最多显示 5 条，底部"查看全部（12条）→"链接

### 5.4 全校心理健康概览

**组件**：Arco 无内置 ECharts，使用 `vue-echarts` 或 `echarts` 直接引入。

**图表类型**：环形图（Donut Chart）

```
数据：
  正常（绿色）：85.2%
  需关注（橙色）：11.5%
  高风险（红色）：3.3%

中心文字：
  "全校"（12px 文字3）
  "1,287人"（20px 600 文字1）
  "已完成测评"（12px 文字3）

右侧图例：
  3 条，每条：色块 12px + 标签 + 占比
```

**交互**：hover 环形扇区，Tooltip 显示具体人数和占比。

### 5.5 近期测评计划进度

**组件**：`a-list`，每条数据使用 `a-progress`（line 类型）

```
每条结构：
  计划名称（13px 500，单行省略）
  班级范围（12px 文字3）
  截止：XX-XX  [a-progress line, strokeWidth=6]  87%
```

### 5.6 今日跟进任务

**组件**：`a-list` + `a-checkbox`

每条结构：
```
□  学生姓名（14px）  —  跟进类型 Tag（面谈/电话/记录）

勾选后：该条灰化 + 文字划线，触发 `Message.success('已完成')`。

## 六、预警管理

### 6.1 页面总体结构

```
[页面标题行]
  标题："预警管理"

[统计卡片行]（3 等分）

## 十九、交互行为规范

### 19.1 加载状态

| 场景 | 组件 |
|------|------|
| 页面初次加载 | `a-skeleton`（模拟内容结构，非 Spin） |

| 图表加载 | ECharts showLoading() |
| AI 内容生成 | `a-skeleton` + 逐字流式展示（SSE）|

### 19.2 操作反馈

| 操作类型 | 反馈 | 持续时间 |
|---------|------|---------|
| 普通保存成功 | `Message.success('保存成功')` | 2s 自动消失 |
| 发布/提交成功 | `Message.success` + 高亮 | 3s |
| 删除成功 | `Message.success` | 2s |
| 表单验证失败 | 字段下方红色错误文字（Arco 默认）| 不自动消失 |
| 系统错误 | `Notification.error`（右上角，含错误码）| 不自动消失 |
| 预警确认成功 | `Notification.success`（内容更丰富）| 5s |
| 批量操作成功 | `Message.success('已处理 X 条')` | 3s |

### 19.3 二次确认规范

以下操作必须使用 `a-modal confirm` 二次确认：

| 操作 | confirm 文案 |
|------|-------------|
| 删除量表 | "删除后已关联的测评计划将不可查看该量表结果，确认删除？" |
| 关闭红色预警 | "关闭红色预警前请确认已完成安全评估并记录，操作将被记录备案" |
| 结案 | "结案后个案将归入已结案状态，可随时重新开案" |
| 取消测评计划 | "已完成作答的数据将保留，未完成的学生将无法继续。确认取消？" |
| 重置密码 | "将发送新密码到用户手机号，确认重置？" |

### 19.4 表单自动保存

测评计划新建页、量表自建页实现自动保存草稿：

## 附录C：需要评审的设计决策

以下设计决策在启动前需产品 + 设计 + 研发三方对齐：

| 决策点 | 当前方案 | 备选方案 | 影响范围 |
|-------|---------|---------|---------|
| H5 作答后是否展示分数 | 不展示 | 展示等级（不展示数字）| 学生端、家长端 |
| AI 解读是否需要老师主动触发 | 自动生成 | 点击按钮触发 | 成本 + 体验 |
| 红色预警关闭是否强制二次密码确认 | 不需要密码，仅 confirm | 需要密码 | 预警管理 |
| 地市看板是否展示学校名称排名 | 展示但脱敏 | 展示明文 | 地市看板、合规 |
| 家长端能否查看孩子每次测评日期 | 可以查看（不含分数）| 仅查看综合状态 | 家长小程序 |
